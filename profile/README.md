# Rule 0 Softworks

Tools for Dungeon Masters who want control, speed, and depth at the table.

Rule 0 Softworks builds modern tooling and content pipelines for tabletop RPGs, focused on D&D 5e and 5.5e, worldbuilding systems, and DM workflow optimization.

---

## What We Do

* DM tooling
  Modular web-based utilities including AI-assisted DM screens, generators, and session tools

* Content pipelines
  Systems for creating and deploying maps, assets, and modules for Roll20 and Foundry

* Worldbuilding systems
  Original settings, mechanics, and language systems such as Librá and Sicco Terra

---

## Active Projects

### Spell Screen

A modular, SRD-compliant DM interface with support for rule lookups, conflict resolution, and session assistance.

Architecture includes a GraphQL and REST backend with a modular frontend UI

---

### AI Batch Runner for Librán

A structured pipeline for generating and evolving fictional languages with consistency and repeatability.

Focus areas include batch processing, linguistic cohesion, and DM-facing usability.

---

### Random SRD Item Generator

A basic interface for rolling individual or bundles of all types of items, armor, and equipment provided by the SRD v5.2.1

---

### D&D Recap Bot

An LLM enhanced Discord bot that takes in a short D&D session description and crafts it into a fully fleshed out recap highlighting key points of the session at the end. Integrates themes for different outcomes. 

---
 
### Rule 0 Content

Marketplace-ready TTRPG assets and modules:

* Maps built with Dungeon Alchemist and custom pipelines
* Token packs and asset bundles
* Modular adventure content

## Tech Approach

* Frontend: Angular with Vercel Deployment for statically served sites
* Backend: Serverless-first, AWS Lambdas
* Data: GraphQL-first, REST where appropriate
* Infrastructure: cost-aware, scalable, secure
* Security: aligned with confidentiality, integrity, availability principles

---

## Design Principles

* Modular architecture over monolithic systems
* Automate repeatable workflows
* SRD-compliant by default
* AI as an augmentation layer
* Prioritize usable systems over theoretical perfection
